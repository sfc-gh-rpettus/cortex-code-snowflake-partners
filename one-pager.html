Memory File Hierarchy
~/.claude/CLAUDE.md
  Global – all projects

~/CLAUDE.md
  Parent – monorepo root

./CLAUDE.md
  Project – shared on git

./frontend/CLAUDE.md
  Subfolder – scoped content
  • Keep each <200 lines
  • Subfolder files append context
  • Never overwrite parent context
4
CLAUDE and Best Practices
  • Run /init first then refine output
  • Be specific in instructions
  • Add gotchas Claude cannot infer
  • Reference docs with @filename
  • Add workflow rules
  • Keep memory concise
  • Commit to Git for team sharing

Adding Skills (The Superpower)

Skills = markdown guides Claude auto-invokes via natural language

Project skill
.claude/skills/<name>/SKILL.md
Personal skill
~/.claude/skills/<name>/SKILL.md
name: testing patterns
description: Jest testing patterns
allowed tools: Read, Grep, Glob
Description field is critical for auto-activation.
Use describe + it + AAA pattern. Use factory mocks.

Skill Ideas:

code-review testing patterns commit messages docker-deploy

Setting Up Hooks

Hooks = deterministic callbacks

PreToolUse PostToolUse Notification
"hooks": {
  "PreToolUse": [{
    "matcher": "Bash",
    "hooks": [{
      "type": "command",
      "command": "scripts/se",
      "timeout": 5
    }]
  }]
}

Exit codes: 0 = allow  |  2 = block
